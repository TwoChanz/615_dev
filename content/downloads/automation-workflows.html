1. Claude Code Automation Patterns

Leverage Claude Code's CLI for powerful development automation.

Auto-Commit with Context

Generate meaningful commit messages based on staged changes.

# Add to your shell profile (.zshrc or .bashrc)
function ai-commit() {
  if ! git diff --cached --quiet; then
    claude -p "Generate a conventional commit message for these staged changes.
    Be concise but descriptive. Format: type(scope): description" \
    --allowedTools "Bash(git diff --cached)"
  else
    echo "No staged changes to commit"
  fi
}

# Usage: stage files, then run
git add .
ai-commit

Code Review Assistant

Get AI-powered code review on your changes before pushing.

# Review current branch changes against main
function ai-review() {
  local base_branch="${1:-main}"
  claude -p "Review the code changes in this branch compared to $base_branch.
  Focus on:
  1. Potential bugs or edge cases
  2. Performance concerns
  3. Security issues
  4. Code style consistency
  Be specific and actionable." \
  --allowedTools "Bash(git diff $base_branch...HEAD)" "Read"
}

# Usage
ai-review main

Quick Bug Fix

Describe a bug and let Claude investigate and fix it.

# Interactive bug fix session
function ai-fix() {
  local description="$1"
  claude "There's a bug: $description

  Please:
  1. Find the relevant code
  2. Identify the root cause
  3. Propose a fix
  4. Apply the fix if I approve"
}

# Usage
ai-fix "Login button doesn't work on mobile Safari"

Test Generator

Auto-generate tests for a specific file or function.

# Generate tests for a file
function ai-test() {
  local file="$1"
  claude -p "Generate comprehensive unit tests for this file.
  Use the project's existing test framework and patterns.
  Include edge cases and error scenarios." \
  --allowedTools "Read" "Write" "Glob" "Grep" \
  "$file"
}

# Usage
ai-test src/lib/auth.ts

2. GitHub Actions Templates

Production-ready CI/CD workflows you can drop into any project.

Next.js CI Pipeline

Complete CI with caching, linting, testing, and build verification.

# .github/workflows/ci.yml
name: CI

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  ci:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v4

      - name: Setup Node
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'

      - name: Install dependencies
        run: npm ci

      - name: Lint
        run: npm run lint

      - name: Type check
        run: npx tsc --noEmit

      - name: Test
        run: npm run test:run

      - name: Build
        run: npm run build
        env:
          NEXT_TELEMETRY_DISABLED: 1

Dependency Updates (Dependabot)

Auto-update dependencies weekly with grouped PRs.

# .github/dependabot.yml
version: 2
updates:
  - package-ecosystem: "npm"
    directory: "/"
    schedule:
      interval: "weekly"
      day: "monday"
    open-pull-requests-limit: 5
    groups:
      dev-dependencies:
        dependency-type: "development"
      production-dependencies:
        dependency-type: "production"
    ignore:
      - dependency-name: "*"
        update-types: ["version-update:semver-major"]

Release Automation

Auto-generate releases with changelogs on version tags.

# .github/workflows/release.yml
name: Release

on:
  push:
    tags:
      - 'v*'

permissions:
  contents: write

jobs:
  release:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Generate changelog
        id: changelog
        uses: orhun/git-cliff-action@v3
        with:
          config: cliff.toml
          args: --latest --strip header

      - name: Create Release
        uses: softprops/action-gh-release@v1
        with:
          body: ${{ steps.changelog.outputs.content }}
          draft: false
          prerelease: ${{ contains(github.ref, 'beta') }}

Preview Deployments Comment

Post deployment URL as PR comment automatically.

# .github/workflows/preview-comment.yml
name: Preview Comment

on:
  pull_request:
    types: [opened, synchronize]

jobs:
  comment:
    runs-on: ubuntu-latest
    steps:
      - name: Wait for Vercel
        uses: patrickedqvist/wait-for-vercel-preview@v1.3.1
        id: vercel
        with:
          token: ${{ secrets.GITHUB_TOKEN }}
          max_timeout: 300

      - name: Comment PR
        uses: actions/github-script@v7
        with:
          script: |
            github.rest.issues.createComment({
              owner: context.repo.owner,
              repo: context.repo.repo,
              issue_number: context.issue.number,
              body: `## Preview Deployment

              Your changes are live at: ${{ steps.vercel.outputs.url }}

              | Check | Status |
              |-------|--------|
              | Build | Passed |
              | Preview | [View](${{ steps.vercel.outputs.url }}) |`
            })

Environment-Specific Builds

Different behavior for preview vs production deployments.

// next.config.ts
const isProduction = process.env.VERCEL_ENV === 'production'
const isPreview = process.env.VERCEL_ENV === 'preview'

const config = {
  // Disable indexing on previews
  ...(isPreview && {
    headers: async () => [
      {
        source: '/:path*',
        headers: [{ key: 'X-Robots-Tag', value: 'noindex' }]
      }
    ]
  }),

  // Only enable analytics in production
  env: {
    NEXT_PUBLIC_ANALYTICS_ENABLED: isProduction ? 'true' : 'false',
  }
}

export default config

4. Notification Automations

Keep your team informed with automated notifications.

Slack Deployment Notifications

Post to Slack when deployments complete.

# .github/workflows/notify-slack.yml
name: Notify Slack

on:
  deployment_status:

jobs:
  notify:
    if: github.event.deployment_status.state == 'success'
    runs-on: ubuntu-latest
    steps:
      - name: Notify Slack
        uses: slackapi/slack-github-action@v1.24.0
        with:
          payload: |
            {
              "blocks": [
                {
                  "type": "section",
                  "text": {
                    "type": "mrkdwn",
                    "text": "*Deployment Successful* :rocket:\n*Repo:* ${{ github.repository }}\n*Branch:* ${{ github.ref_name }}\n*URL:* ${{ github.event.deployment_status.target_url }}"
                  }
                }
              ]
            }
        env:
          SLACK_WEBHOOK_URL: ${{ secrets.SLACK_WEBHOOK_URL }}
          SLACK_WEBHOOK_TYPE: INCOMING_WEBHOOK

Email Alerts for Failures

Get emailed when CI fails on main branch.

# .github/workflows/alert-failure.yml
name: Alert on Failure

on:
  workflow_run:
    workflows: ["CI"]
    types: [completed]
    branches: [main]

jobs:
  alert:
    if: github.event.workflow_run.conclusion == 'failure'
    runs-on: ubuntu-latest
    steps:
      - name: Send failure alert
        uses: dawidd6/action-send-mail@v3
        with:
          server_address: smtp.resend.com
          server_port: 465
          username: resend
          password: ${{ secrets.RESEND_API_KEY }}
          subject: "CI Failed: ${{ github.repository }}"
          to: alerts@yourcompany.com
          from: GitHub Actions <ci@yourcompany.com>
          body: |
            CI pipeline failed on main branch.

            Commit: ${{ github.event.workflow_run.head_sha }}
            Author: ${{ github.event.workflow_run.head_commit.author.name }}

            View run: ${{ github.event.workflow_run.html_url }}

Daily Digest Notification

Summarize repository activity and send daily report.

# .github/workflows/daily-digest.yml
name: Daily Digest

on:
  schedule:
    - cron: '0 17 * * 1-5'  # 5 PM UTC, weekdays

jobs:
  digest:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Generate digest
        id: digest
        uses: actions/github-script@v7
        with:
          script: |
            const yesterday = new Date(Date.now() - 86400000).toISOString()

            const commits = await github.rest.repos.listCommits({
              owner: context.repo.owner,
              repo: context.repo.repo,
              since: yesterday
            })

            const prs = await github.rest.pulls.list({
              owner: context.repo.owner,
              repo: context.repo.repo,
              state: 'all',
              sort: 'updated',
              direction: 'desc'
            })

            return {
              commits: commits.data.length,
              prs_opened: prs.data.filter(p =>
                new Date(p.created_at) > new Date(yesterday)
              ).length,
              prs_merged: prs.data.filter(p =>
                p.merged_at && new Date(p.merged_at) > new Date(yesterday)
              ).length
            }

      - name: Send to Slack
        uses: slackapi/slack-github-action@v1.24.0
        with:
          payload: |
            {
              "text": "Daily Digest for ${{ github.repository }}: ${{ steps.digest.outputs.result }}"
            }
        env:
          SLACK_WEBHOOK_URL: ${{ secrets.SLACK_WEBHOOK_URL }}

Useful Claude Code Flags

claude -p "prompt"              # Non-interactive (pipe mode)
claude --allowedTools "Read"    # Restrict tools
claude --model sonnet           # Use specific model
claude --output-format json     # JSON output
claude --max-turns 5            # Limit conversation turns
Security Note: Never commit secrets to your repository. Use GitHub Secrets, Vercel Environment Variables, or a secrets manager. Rotate credentials if accidentally exposed.
