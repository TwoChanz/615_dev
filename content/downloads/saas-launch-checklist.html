SaaS Launch Checklist

The complete pre-launch checklist for indie hackers and solo founders

1. Pre-Development

Validation

  • Problem statement documented (who, what problem, why it matters)
  • Talked to 5+ potential customers about the problem
  • Competitor analysis completed (features, pricing, gaps)
  • Unique value proposition defined in one sentence
  • MVP scope defined (core features only, no nice-to-haves)

Business Foundation

  • Domain name purchased
  • Brand name validated (trademark search, social handles available)
  • Pricing strategy decided (freemium, trial, paid-only)
  • Target customer persona documented

2. Development Phase

Technical Setup

  • Git repository initialized with proper .gitignore
  • Environment variables structure defined (.env.example)
  • CI/CD pipeline configured (GitHub Actions, Vercel, etc.)
  • Error monitoring set up (Sentry or similar)
  • Analytics integrated (PostHog, Plausible, or GA)

Core Features

  • User authentication working (sign up, login, logout, password reset)
  • Core value feature functional and tested
  • Onboarding flow guides new users
  • Settings page allows profile updates
  • Data can be exported (GDPR compliance)

Payments

  • Stripe account verified and connected
  • Products and prices created in Stripe
  • Checkout flow working (test mode)
  • Webhook handling subscription events
  • Customer portal accessible for plan management
  • Invoice emails configured
Pro Tip: Use Stripe's test clock feature to simulate subscription renewals and test your webhook handling before going live.

3. Pre-Launch Essentials

Legal & Compliance

  • Privacy Policy published
  • Terms of Service published
  • Cookie consent banner (if applicable)
  • GDPR data deletion request process defined
  • Business entity formed (LLC, etc.) - optional but recommended

SEO & Marketing Site

  • Landing page with clear value proposition
  • Feature/benefit sections
  • Pricing page with FAQ
  • Meta tags and Open Graph images
  • Sitemap.xml generated
  • Google Search Console connected

Support Infrastructure

  • Contact form or support email configured
  • FAQ/Help documentation started
  • Response time expectations set

4. Launch Week

Final Testing

  • Full user journey tested (signup → payment → core feature → cancel)
  • Mobile responsiveness verified
  • Load testing completed (basic)
  • All environment variables set in production
  • Database backups configured
  • SSL certificate active

Go Live

  • Stripe switched to live mode
  • DNS propagated
  • Monitoring dashboards open
  • First payment tested with real card

5. Post-Launch (Week 1)

Distribution

  • Product Hunt launch scheduled/completed
  • Hacker News Show HN posted
  • Reddit relevant subreddits posted
  • Twitter/X announcement thread
  • LinkedIn post for professional network
  • Indie Hackers milestone posted

Feedback Loop

  • User feedback collection method active
  • Bug reports triaged and prioritized
  • First customer testimonial requested
  • Analytics reviewed for drop-off points
Remember: Launch is just the beginning. The real work starts when you have paying customers. Focus on retention before acquisition.
